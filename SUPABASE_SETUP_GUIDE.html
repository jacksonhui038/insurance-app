Supabase 設定指南 — 保經管家

呢份指南會帶你一步一步建立 Supabase 數據庫，同埋將保經管家嘅數據安全遷移過去。

預計時間： 15 分鐘 難度： 簡單（跟住做就得）費用： 免費（500MB 額度）

第一步：開 Supabase 帳戶

1️⃣ 註冊

  1. 去 https://supabase.com
  2. 撳右上角 "Start your project"
  3. 用 GitHub 帳號登入（最簡單，唔使填表）

或者直接用 Google 帳號登入都得。

2️⃣ 建立新專案

  1. 登入後撳 "New project"
  2. 填寫以下資料：
    欄位填咩
    Nameinsurance-manager（或任何名）
    Database Password設定一個密碼（記低佢！）
    RegionNortheast Asia (Tokyo) — 最近香港
    Pricing PlanFree（免費 500MB）
  3. 撳 "Create new project"
  4. 等 1-2 分鐘建立完成

第二步：建立資料表

3️⃣ 開啟 SQL Editor

喺左邊 sidebar 撳 SQL Editor（第二個 icon）→ 撳 "New query"

4️⃣ 貼上 SQL 並執行

複製下面全部 SQL，貼入 SQL Editor，撳右下角綠色 "Run" 掣：

-- 代理人數據表
CREATE TABLE IF NOT EXISTS agents (
  id        BIGSERIAL PRIMARY KEY,
  name      TEXT NOT NULL,
  data      JSONB NOT NULL DEFAULT '{}',
  manager   TEXT DEFAULT '',
  team      TEXT DEFAULT '',
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_agents_name ON agents(name);
CREATE INDEX IF NOT EXISTS idx_agents_updated ON agents(updated_at DESC);

-- 自動更新時間
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_agents_updated_at
  BEFORE UPDATE ON agents
  FOR EACH ROW
  EXECUTE FUNCTION update_updated_at();

-- 啟用 RLS
ALTER TABLE agents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow all for anon" ON agents
  FOR ALL USING (true) WITH CHECK (true);
見到 "Success. No rows returned" 就代表成功！

第三步：攞 API Key

5️⃣ 複製網址同 Key

  1. 左邊 sidebar 撳 Settings（齒輪 icon，最底）
  2. 撳 API（喺左邊子選單）
  3. 你會見到兩個重要資料：
    欄位喺邊搵
    Project URLhttps://xxxxxxxxxxxx.supabase.co
    anon public key好長一串，以 eyJhbGciOi... 開頭
⚠️ 記得用 anon/public key，唔好用 secret key！Secret key 係俾 server 用嘅，放喺前端會好危險。

第四步：設定保經管家

6️⃣ 喺代理 App 設定

  1. 打開代理人版
  2. 登入後撳右上角 ☁️ 雲端
  3. 喺 "Supabase（推薦）" 區域：
    • 專案網址：貼上 Project URL（例如 https://abc123.supabase.co）
    • anon key：貼上 anon public key
  4. 撳 "儲存設定"
  5. 系統會自動同步現有數據到 Supabase ✅

7️⃣ 喺經理版設定

  1. 打開經理版
  2. 登入後撳 ⚙️ 設定
  3. 貼上同樣嘅 Project URL + anon key
  4. 撳 "儲存並載入" → 會自動讀取團隊數據

雙寫安全機制（已經內建）

✅ Google Sheets 同 Supabase 可以同時用！

你唔需要即刻放棄 Google Sheets。系統已經內建「雙寫」機制：
  • 每次同步會同時寫入 Supabase + Google Sheets
  • 載入數據優先 Supabase、fallback Google Sheets
  • 任何一邊出事，另一邊繼續運作
  • 確認 Supabase 穩定後，可以喺設定入面清空 Google Sheets 網址

常見問題

Q: 換返 Google Sheets 會唔會好麻煩？

A: 完全唔麻煩。 去雲端設定清空 Supabase 欄位，留返 Google Sheets 網址就得。所有代理人手機 localStorage 嘅數據唔會消失，撳「同步」就推返上去。

Q: 免費 500MB 夠幾多個代理人用？

A: 好夠。 每個代理人嘅全部數據（客戶 + 保單 + 佣金 + 行程 + 收入 + 支出 + PC）大約 50-200KB。即係 500MB 夠 2,500-10,000 個代理人用。

Q: 數據安唔安全？

A: 好安全。 Supabase 有 SOC2 + GDPR 認證，傳輸加密（HTTPS），靜態加密（encryption at rest），Row-Level Security。比 Google Sheets API URL 安全好多。

Q: 需要信用卡嗎？

A: 唔需要。 Free plan 唔使信用卡，永久免費。超額先會問你升級。
